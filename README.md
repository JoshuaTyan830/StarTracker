# StarTracker：歷年繁星推薦數據分析與落點分析平台

## Proposal Report

### 動機與目標
繁星推薦志願選填時，歷年錄取標準（校排%、學測門檻）散落在各年度獨立網頁中，導致跨年份、跨校系的趨勢比對極度耗時且難以精確量化。本專案的目標是整合近 10 年結構化數據，打造一個具備智慧換算功能的分析平台。透過「累積人數百分比」與「時間權重演算法」，將歷史數據轉化為具參考價值的落點預測，協助學弟妹從海量資料中快速篩選最佳志願。

### 預期功能
1. **歷年趨勢視覺化**：動態呈現單一校系近 10 年錄取百分比及學測級分變動折線圖。
2. **跨校系平行對比**：支援選取多個校系進行橫向對比，一目了然各校系競爭強度的消長。
3. **個人化智慧落點分析**：
    * **等值級分換算**：根據當年度與歷年的人數百分比區間，自動計算不同年份間的「等值級分」。
    * **加權預測模型**：引入時間權重（如：愈近年度權重愈高），計算各校系之推薦指數。
4. **動態志願序試排**：提供拖拉式清單介面，讓使用者自訂志願順序並即時進行風險評估。
5. **純本地端安全運算**：所有成績輸入與分析皆於瀏覽器端完成，確保使用者個資（成績）絕不外傳至伺服器。

### 使用技術
* **資料工程**：使用 Python/C++ 進行歷年原始簡章與分發結果的清洗（Cleaning）與 JSON 結構化。
* **網頁前端**：React.js 搭配 Tailwind CSS 構建響應式介面。
* **數據視覺化**：Chart.js 或 Recharts 實作動態圖表。
* **部署平台**：GitHub Pages (採靜態網頁架構以極小化資安風險)。

### 時程規劃
* **Week 8-9**：完成 10 年份繁星原始資料抓取與清洗，產出核心 `data.json`。
* **Week 10-11**：開發 C++/JS 核心換算演算法與資料檢索邏輯，建立專案 Prototype 。
* **Week 12-14**：實作前端 UI/UX 介面、圖表視覺化整合及志願序拖拉功能。
* **Week 15**：壓力測試、錄製 Demo 影片並發布 Final Report。

### 與課程的關聯
本專案的底層資料處理與檢索，將面臨以下幾項核心需求，這與課程中探討的資料結構與演算法概念高度相關：
1. **海量校系資料的極速檢索需求**：系統需要載入並處理全台數千個校系長達 10 年的歷史數據。當使用者查詢特定校系時，系統必須能無延遲地給出對應的歷史資料，這涉及了如何利用「鍵值對」特性來達成 $O(1)$ 搜尋時間的資料結構（如 Hash Table）。
2. **歷史數據的區間比對需求**：在進行「智慧落點分析」時，系統需要將使用者的成績，與歷年龐大的「累積人數百分比」陣列進行比對，找出準確的落點區間。這對應到課程中對於已排序資料進行高效搜尋的演算法設計（如 Binary Search）。
3. **流暢的使用者輸入體驗需求**：為了讓使用者在搜尋框輸入部分字首，就能快速跳出校系全名提示（Auto-complete），系統需要有效率地處理字串前綴匹配，這與特定樹狀結構（如 Trie）的應用場景完全契合。
4. **多維度數據的動態排序需求**：系統允許使用者依據「錄取機率」、「學測級分」或「加權權重」等多種指標，重新排列他們的志願清單。如何針對這些包含多種屬性的結構化資料，選擇並實作最適合的排序演算法以達到最佳效能，是本專案的核心挑戰。
