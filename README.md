# StarTracker：歷年繁星推薦數據分析與落點分析平台

## Proposal Report

### 動機與目標
繁星推薦志願選填時，歷年錄取標準（校排%、學測門檻）散落在各年度獨立網頁中，導致跨年份、跨校系的趨勢比對極度耗時且難以精確量化。本專案的目標是整合近 10 年結構化數據，打造一個具備智慧換算功能的分析平台。透過「累積人數百分比」與「時間權重演算法」，將歷史數據轉化為具參考價值的落點預測，協助學弟妹從海量資料中快速篩選最佳志願。

### 競品比較
目前市面上的繁星落點分析工具（如：交叉查榜）雖然具備基礎的條件篩選功能，但仍有以下顯著缺點：
1. **落點預測過於籠統**：分析結果缺乏精確的機率量化，僅粗略分為「有機會通過」與「不確定會通過（夢幻科系，別忘了選）」兩種極端分類，無法提供學生更細緻的風險評估與志願排序依據。
2. **歷年標準排版糟糕且缺乏視覺化**：點擊查看「最近三年錄取標準」等歷史數據時，介面僅以生硬且排版擁擠的純文字條列，閱讀體驗極差；同時缺乏直觀的圖表呈現，使用者難以察覺分數或錄取率的長期波動趨勢。
3. **整體使用者體驗（UX）不佳**：介面設計停留在早期網頁風格，版面充斥大量廣告區塊與繁瑣的表單勾選，不僅干擾視覺，在跨校系對比時也需要頻繁切換頁面，操作不夠直覺。
4. **個資隱私疑慮**：傳統平台多採用伺服器端運算，使用者的學測成績、在校成績等隱私數據必須回傳至遠端主機。

**StarTracker 的核心優勢**：主打「精準量化的加權預測」、「現代化響應式介面」與「動態圖表視覺化」，並採用「純前端本地運算」架構，確保零延遲體驗的同時完全保障使用者成績隱私。

### 預期功能
1. **歷年趨勢視覺化**：動態呈現單一校系近 10 年錄取百分比及學測級分變動折線圖。
2. **跨校系平行對比**：支援選取多個校系進行橫向對比，一目了然各校系競爭強度的消長。
3. **個人化智慧落點分析**：
    * **等值級分換算**：根據當年度與歷年的人數百分比區間，自動計算不同年份間的「等值級分」。
    * **加權預測模型**：引入時間權重（如：愈近年度權重愈高），計算各校系之推薦指數。
4. **動態志願序試排**：提供拖拉式清單介面，讓使用者自訂志願順序並即時進行風險評估。
5. **純本地端安全運算**：所有成績輸入與分析皆於瀏覽器端完成，確保使用者個資（成績）絕不外傳至伺服器。

### 使用技術
* **資料工程**：使用 Python 進行歷年原始簡章與分發結果的清洗（Cleaning）與 JSON 結構化。
* **網頁前端**：React.js 搭配 Tailwind CSS 構建響應式與現代化介面。
* **數據視覺化**：Chart.js 或 Recharts 實作動態趨勢圖表。
* **部署平台**：GitHub Pages (採靜態網頁架構以極小化資安風險)。

### Prototype 預計可驗證內容 (Week 11 檢核點)
為確保專案核心邏輯與技術架構具備可行性，Prototype 階段預計完成並驗證以下項目：
1. **結構化資料庫建置**：展示由 Python 自動化爬蟲與資料清洗管線所產出的高質量 JSON 核心資料檔（涵蓋至少單一學年度的完整校系檢定標準與名額）。
2. **極速檢索與前端串接**：完成基礎 React 網頁雛形，驗證能將龐大 JSON 資料載入前端，並實作透過校系名稱或代碼即時檢索、渲染結果的功能。
3. **單一年度落點條件篩選器**：實作基礎比對演算法，使用者輸入特定學測級分與在校成績後，系統能準確篩選出符合「第一階段檢定標準」的校系清單，驗證核心邏輯的正確性。

## Prototype Report

### 目前進度
*   **資料庫建置：** 已完成 112 至 115 學年度繁星推薦錄取標準資料的清洗與格式化（JSON 格式）。
*   **前端介面雛形：** 成功建置基於 React + Vite + Tailwind CSS v4 的現代化前端展示面板（Prototype）。
*   **核心功能實作：**
    *   實作全端搜尋列（支援校系代碼、校名、系名關鍵字搜尋）與學群下拉篩選器。
    *   建立校系代碼對照表（Mapping），自動將代碼轉換為完整的學校名稱。
    *   完成「歷年錄取標準比序」Modal 彈出視窗，將官方 PDF 中複雜的檢定標準（轉為視覺化標籤）與分發比序結果（轉為清晰表格）進行完美重構與並排比較。
    *   於 UI 介面中成功預留並設計「智慧級分轉換」的視覺化預覽元件（Mockup）。

### 遇到的困難
*   **資料去重邏輯「誤殺」外加名額：**
    *   *問題：* 在清理 JSON 資料時，發現「一般生」與「外加名額」會共用同一個校系代碼（如森林系與森林系【外加】皆為 `00151`），導致去重程式將其視為重複資料而覆蓋。
    *   *解決方式：* 引入資料庫工程中的「複合鍵（Composite Key）」概念，將「代碼 + 系所名稱」綁定作為唯一識別碼，成功保留所有獨立名額。
*   **前端樣式框架（Tailwind CSS）版本衝突：**
    *   *問題：* 專案初始化時遭遇 Tailwind CSS 最新 v4 版本的底層架構大改版，導致舊有的 PostCSS 設定檔寫法失效且不斷報錯。
    *   *解決方式：* 查閱最新官方文件，全面升級為 Vite 專屬的 `@tailwindcss/vite` 插件架構，並執行物理刪除殘留設定檔與清除 Vite 暫存（`--force`），成功排除環境報錯。
*   **版本控制（Git）推送衝突：**
    *   *問題：* 嘗試將完成的 Prototype 推送至 GitHub 時，因遠端倉庫存在本地端缺乏的更新檔案，導致 `git push` 被拒絕（Rejected）。
    *   *解決方式：* 執行 `git pull origin main` 將雲端進度拉取下來並完成合併（Merge）後，再次推送即順利完成備份。

### 下一步計畫
*   **開發級分轉換演算法：** 載入 `gsat_historical_stats.json`（歷年學測累積人數統計資料），撰寫「累積人數換算函數」，以推算不同年份間等值的級分落點。
*   **串接動態數據：** 將算好的級分轉換函數與前端 Modal 介面正式串接，取代現有的靜態預覽徽章，實現真實的跨年份級分轉換對照功能。
*   **細節優化：** 持續微調前端 UI 排版細節與使用者體驗（UX），確保系統在展示時的流暢度與穩定度，為期末的 Final Report 做好準備。
